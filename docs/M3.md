# M3 — OpenCode handshake + embed

M3 wires OpenCode inside the container to the loom Worker: the embed
renders correctly, the MCP server accepts platform JWTs from the
sandbox, and OpenCode authenticates with a model provider on first
launch.

---

## What M2 got wrong

M2 attempted to proxy OpenCode's full HTML/CSS/JS through the Worker at
`/dash/oc/*` using `createOpencodeServer` + `proxyToOpencodeServer`.
This fails because OpenCode's Vite build bakes **root-relative** asset
paths (`/assets/index-BLeT4xV9.js`) into the HTML at build time. When
the HTML is served under `/dash/oc/`, the browser fetches assets from
the Worker root (`/assets/...`) which returns 404. HTML rewriting is
fragile and wrong.

The correct approach — established by the `let-it-slide` reference
implementation — is to build OpenCode's frontend as a **static mount
bundle** served as Worker assets and proxy only the API calls to the
container. The container's HTML is never sent to the browser.

M3 replaces the M2 `/dash/oc/*` implementation entirely.

---

## Architecture

```
Browser iframe
  └─ /opencode-ui/embed.html?serverUrl=/opencode/<userId>&directory=/home/user/workspace
       │
       ├─ loads /opencode-ui/opencode-mount.js    (static Worker asset)
       ├─ loads /opencode-ui/opencode-mount.css   (static Worker asset)
       └─ calls mount(root, { serverUrl, directory })
                │
                └─ all API calls → /opencode/<userId>/api/...
                                        │
                                Worker: strips /opencode/<userId>
                                        │
                                container port 4096 (/api/...)
```

OAuth flow:
```
OpenCode (in container) initiates auth
  → redirectUri = https://loom.cloudemo.org/opencode-oauth/<userId>/callback
  → user authenticates in browser
  → browser hits /opencode-oauth/<userId>/callback
  → Worker (Access-gated) strips prefix, proxies to container OAuth listener
  → container stores credentials in ~/.opencode/auth.json
```

### D1 — Build the OpenCode mount bundle

Add a `prebuild` step that clones OpenCode at a pinned commit, injects
a custom entry point, and builds a library bundle with
`base: "/opencode-ui/"` so all asset paths are root-relative from that
prefix (not `/`).

**New files:**

`apps/web/public/opencode-ui/VERSION`
```
REPO=https://github.com/anomalyco/opencode
COMMIT=<pinned commit SHA>
```

`apps/web/public/opencode-ui/embed.html`
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>OpenCode</title>
  <link rel="stylesheet" href="/opencode-ui/opencode-mount.css" />
  <style>
    html, body, #root { margin: 0; padding: 0; width: 100%; height: 100%; overflow: hidden; }
    #root { display: flex; flex-direction: column; }
    body[data-opencode-embedded] #terminal-panel { display: none !important; }
  </style>
</head>
<body data-opencode-embedded="true">
  <div id="root"></div>
  <script type="module">
    const params = new URLSearchParams(window.location.search);
    const serverUrl = params.get("serverUrl");
    const directory = params.get("directory") || "/home/user/workspace";
    const sessionId = params.get("sessionId");
    const parentOrigin = window.location.origin;

    if (!serverUrl) {
      window.parent.postMessage({ type: "opencode-embed-error", error: "Missing serverUrl" }, parentOrigin);
    } else {
      import("/opencode-ui/opencode-mount.js")
        .then(({ mount }) => {
          mount(document.getElementById("root"), { serverUrl, directory, sessionId });
          window.parent.postMessage({ type: "opencode-embed-ready" }, parentOrigin);
        })
        .catch((err) => {
          window.parent.postMessage({ type: "opencode-embed-error", error: err.message }, parentOrigin);
        });
    }
  </script>
</body>
</html>
```

`scripts/opencode-mount/mount.tsx` — custom SolidJS entry point (adapt
from let-it-slide). Takes `serverUrl`, sets the hash route for the
directory, and calls OpenCode's `render()` with an HTTP server
connection pointed at `serverUrl`.

`scripts/opencode-mount/vite.mount.config.ts` — Vite library build
config. Key settings:
```typescript
base: "/opencode-ui/",
build: {
  lib: { entry: "src/mount.tsx", formats: ["es"], fileName: "opencode-mount" },
  outDir: "dist-mount",
}
```

`scripts/build-opencode-ui` — bash script (run as `prebuild`):
1. Read `REPO` and `COMMIT` from `apps/web/public/opencode-ui/VERSION`
2. Clone or fetch `REPO` into `.cache/opencode`, checkout `COMMIT`
3. Copy `scripts/opencode-mount/mount.tsx` and
   `scripts/opencode-mount/vite.mount.config.ts` into the clone
4. Install with bun, build with `bun x vite build --config vite.mount.config.ts`
5. Copy `dist-mount/` → `apps/web/public/opencode-ui/` (preserving
   `embed.html` and `VERSION`)
6. Write a `.build-stamp` so re-runs skip if commit and inputs are unchanged

### D2 — New proxy routes in worker-entry.ts

Replace `/dash/oc/*` with two new routes. Both run before the generic
`/dash/*` handler.

**`/opencode/<userId>/...`** (OpenCode API proxy — Access-gated):
```
GET /opencode/<userId>/api/session.list
  → auth check (Access JWT) → verify userId matches token
  → strip /opencode/<userId>
  → container:4096/api/session.list
```

Implementation:
```typescript
if (url.pathname.startsWith("/opencode/")) {
  const auth = await getAuthContext(request, env);
  const [, , sandboxId, ...rest] = url.pathname.split("/");
  if (sandboxId !== auth.userId) return new Response("Forbidden", { status: 403 });

  const sandbox = getSandbox(env.SANDBOX, auth.userId, { keepAlive: true, normalizeId: true });
  const targetUrl = new URL("/" + rest.join("/") + url.search, "http://localhost:4096");
  const proxied = new Request(targetUrl, {
    method: request.method,
    headers: request.headers,
    body: request.body,
    redirect: "manual",
  });
  return sandbox.containerFetch(proxied, 4096);
}
```

**`/opencode-oauth/<userId>/...`** (OAuth callback — also Access-gated):

OpenCode's OAuth flow redirects the browser to this path after the user
authenticates with `opencode.cloudflare.dev`. The Worker catches it,
verifies the outer Access JWT, and proxies the full path (including
everything after `/<userId>`) to the container's in-process OAuth
listener port.

```typescript
if (url.pathname.startsWith("/opencode-oauth/")) {
  const auth = await getAuthContext(request, env);
  const [, , sandboxId] = url.pathname.split("/");
  if (sandboxId !== auth.userId) return new Response("Forbidden", { status: 403 });

  const sandbox = getSandbox(env.SANDBOX, auth.userId, { keepAlive: true, normalizeId: true });
  // Forward the full /opencode-oauth/<userId>/... pathname to the container's
  // OAuth callback listener (OpenCode binds to the redirectUri port).
  const oauthPort = url.port ? parseInt(url.port) : (url.protocol === "https:" ? 443 : 80);
  const proxied = new Request(new URL(url.pathname + url.search, `http://localhost:${oauthPort}`), {
    method: request.method,
    headers: request.headers,
    body: request.body,
    redirect: "manual",
  });
  return sandbox.containerFetch(proxied, oauthPort);
}
```

**Remove** `proxyOcRequest`, `createOpencodeServer`, `proxyToOpencodeServer`,
and the HTML-rewriting block from M2.

Iframe element:

The Worker no longer needs to start OpenCode itself at iframe-load time.

### D4 — Platform JWT and opencode.jsonc injection

On the first request to `/opencode/<userId>/...`, before proxying:

1. **Mint a platform JWT**:
   ```
   { sub: userId, session_id: crypto.randomUUID(), iat, exp: iat + 7200 }
   ```
   Signed with HMAC-SHA256 using `PLATFORM_JWT_SECRET`.

2. **Write opencode.jsonc into the container** via a small setup process
   run once per container boot. The Worker calls
   `sandbox.startProcess("loom-setup", { env: { ... } })` or writes the
   file via a container exec before starting OpenCode:
   ```jsonc
   {
     "$schema": "https://opencode.ai/config.json",
     "autoupdate": false,
     "server": { "port": 4096 },
     "mcp": {
       "loom": {
         "type": "remote",
         "url": "https://<LOOM_HOSTNAME>/mcp",
         "headers": { "Authorization": "Bearer <platform_JWT>" }
       }
     }
   }
   ```

3. **Start OpenCode** (if not already running):
   ```
   opencode serve --port 4096 --hostname 0.0.0.0
   ```

The session-scoped platform JWT is stored in the `UserRegistry` DO's
session table so the Worker can verify it on subsequent `/mcp` calls
from the container.

**`/mcp` handler update**: accept either an Access JWT
(`Cf-Access-Jwt-Assertion` header) or a platform JWT (`Authorization:
Bearer <token>` header). Derive `userId` from whichever is present.

### D5 — Dockerfile simplification

Remove `USER user` and the binary copy/symlink. Let-it-slide runs as
root — the Sandbox SDK manages container lifecycle regardless of the
Unix user. This eliminates the permission problems that plagued M2.

```dockerfile
# Remove: USER user
# Remove: RUN id -u user ... || useradd ...
# Remove: cp ... /usr/local/bin/opencode && chmod ...

# Replace with:
ENV PATH="/root/.opencode/bin:${PATH}"

RUN curl -fsSL https://opencode.ai/install | bash && opencode --version
```

The workspace directory is still `/home/user/workspace` (OpenCode's
default convention) but the process runs as root, which has access to
`/root/.opencode/` where the binary and credentials live.
